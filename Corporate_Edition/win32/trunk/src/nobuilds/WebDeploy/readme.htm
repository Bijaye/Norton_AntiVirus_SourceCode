Overview

    WebInstall allows for easy deployment of Windows MSI based software from a Web server. WebInstall is capable of installing many applications with a single click! Since WebInstall is HTML based it allows for easy rebranding, easy installation, and is compliant with all HTTP web servers on all platforms.

      Note: The server side component of WebInstall does not require a Windows platform. The client does require the Windows platform. 

    WebInstall allows administrators to control the installation sequence and how installations are to be performed. For example, an administrator may want to make an installation silent with no user interaction. MSI installation controls are specified in the configuration file. The controls should conform to Microsoft's MSI command line standard when executing MSIExec.

    WebInstall works by installing an ActiveX control on the client's machine. The control is automatically installed when a user starts the installation processes. The ActiveX control will only be downloaded when it's not installed on the client machine. Typically, the control will only need to be downloaded once. Once installed the ActiveX control is passed the configuration file (files.ini) that controls the sequence of events. The configuration file contains all the information needed to download file(s) and execute a single file. The executable file then executes the MSIExec command to install the application(s).

    WebInstall follows the following sequence when executed:
    • Download ActiveX control (if needed).
    • Download configuration file.
    • Download all files specified in the configuration file.
    • Execute the file specified by the LaunchApplication tag in the configuration file.
Installation

Deploying software using WebInstall consists of the following steps:

  • Copy required files (supplied with WebInstall) to the web server. Required files consists of:
    • files.ini
    • launch.bat
    • brnotsup.htm
    • default.htm
    • intro.htm
    • logo.jpg
    • oscheck.htm
    • plnotsup.htm
    • start.htm
    • webinst.cab
    • "... Any number of MSI files."
  • Edit 'files.ini' adding any files that need to be installed.
  • If unique branding is required perform the following:
    • Replace logo.jpg with the appropriate logo.
    • Edit text in HTML pages to reflect desired branding.
  • Create a link to WebInstall's HTML installation page (intro.htm).
  • Test the link with a compatible browser.
Requirements & Limitations
    Server Requirements
  • HTTP Web Server.
    Client Requirements
  • Microsoft Internet Explorer version 5.5 Service Pack 2 or higher.  Installing Symantec Client Security or Symantec AntiVirus Corporate Edition requires Microsoft Internet Explorer 5.5 Service Pack 2 or later.
  • User must allow ActiveX controls to be installed on local machine.
    •  

    Limitations

  • No authentication is required to download and install software.
  • Files cannot be hosted on multiple web servers.
  • Files must reside on the server in the same directory with, or in a relative path from, the webinstall directory.
  • WebInstall executes a single file (Use a batch file to execute multiple files).
  • File and path names are case sensitive on Unix machines.
