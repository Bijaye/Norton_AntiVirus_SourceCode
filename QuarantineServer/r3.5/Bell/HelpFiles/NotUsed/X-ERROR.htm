X-Error Attribute

The proprietary X-Error attribute is included in response messages when requests cannot be processed correctly. They describe the reason that the request could not be processed, and may optionally include parameters that further qualify the error:

X-Error: error code and optional parameters

For example, a server that is unable to process a signature download request for a set of signatures named "AV30BA" because they are no longer available might include this header in its response:

     X-Error: superceded

For another example, a server that is unable to process a sample submission request because the value of the "Content-Length" header does not match the size of the message content might include this header in its response:

     X-Error: overrun 10138 10139

Error Codes

abandoned A signature sequence number specified as an argument has been abandoned, either because unit-testing of the corresponding definition set has failed, or because a lock on the source library or build system acquired by the back-office gateway has been broken.
content The checksum of the content of the request message did not match the value of the X-Content-Checksum attribute. The actual checksum and expected checksum are parameters, represented as hexadecimal numbers. This may be due to a transmission error within the transport network.
crumbled The cookie specified as an argument in the request has not been assigned by the gateway. This is probably due to a programming error in the client.
declined The sample submitted for analysis has been declined by the gateway. This may be due to invalid service subscriber identification or over-use of the service. However, no information about the reason for declining the sample is returned to the client. Instead, the client should instruct the user to contact technical support for assistance.
internal An internal error occurred while processing a sample.
lost The sample was not completely received due to a network failure.
malformed An essential attribute in a request message was malformed. The attribute name and erroneous value are included as parameters. This is usually due to a programming error in the client or the server.
missing An essential attribute in a request message was missing. The attribute name is included as a parameter. This is usually due to a programming error in the client or the server.
overrun The length of the content of the request message is greater than the Content- Length header. The actual length and expected length are parameters, represented as decimal numbers. This may be due to a transmission error in the transport network, but is more likely due to a programming error in the client, perhaps in sending binary data containing newline characters through a subroutine library as text data.
sample The checksum of the sample did not match the value of the X-Sample-Checksum attribute. The actual checksum and expected checksum are parameters, represented as hexadecimal numbers. This may be due to a programming error in the client or server, perhaps in calculating a checksum or in compressing a sample file.
superceded The signatures named as an argument in the request have been superceded by newer blessed signatures and are no longer available from the server. The client should download the current blessed signatures instead of the superceded signatures.
type The sample type specified by the X-Sample-Type attribute is not supported. The unsupported value is a parameter.
underrun The length of the content of the request message is less than the Content-Length header. The actual length and expected length are parameters, represented as decimal numbers. This may be due to a transmission error in the transport network, but is more likely due to a programming error in the client or server, perhaps in calculating the length of compressed content.
unavailable The signature set specified as an argument is not yet available on the server. The client should wait for a pre-configured interval and then re-request the signature set.
